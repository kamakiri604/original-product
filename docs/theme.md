# 解決したい課題 / 潜在的なニーズ

- 食材費の高騰に対し、どのメニューを、いつ、いくら値上げするまたは仕様変更すべきかなどの意思決定が遅れ、利益を把握しにくくなっている。
- 異常値をリアルタイムに検知し、オーナーがそれに対するアクションの判断を助ける。

# 対象ユーザー（誰の、どんな状況か）

- 飲食店のオーナーや管理者。
- パソコンをじっくり開く時間はなく、キッチンの現場で仕込みをしながら、手元のスマートフォンでサクッと原価や利益を確認したいと考えている人。
- ITツールにあまり詳しくなく、マニュアルを読むのが苦手な仕込みに忙しい人。

# プロダクトのカテゴリー

- 食材原価計算・利益管理ツール

# 提供する価値（ユーザーが得られる変化・体験）

- 高騰したメニューが一目でわかり、赤字を防げる
- 仕入れ値を変えるだけで全てのメニューの原価率が連動して変わるため、Excelの修正作業から解放されます。
- 目標原価率を超えたメニューには自動でアラートが出るため、価格改定やレシピ見直しのタイミングを逃さなくなる。

# 価値を実現するための最小限の機能（MVP）

- 食材名、仕入れ価格、内容量を登録すると、1gあたりの単価を自動算出。
- メニュー名と販売価格を設定し、使用する食材とグラム数を紐づけ。減価率を算出。
- 食材の価格変更時に全レシピの原価率を更新。目標原価率を超えた場合警告を表示。

# 既存の代替手段・競合との違い

- Excel・紙のメモとの違い：スマホ画面に完全最適化。忙しくパソコンでの作業に時間が取られたくない人のために、片手で直感的に食材の追加や数量変更ができる。
- 大手の業務用システムとの違い：「発注・在庫・シフト管理」といった複雑な機能をあえてすべて削ぎ落とし、「食材を登録してレシピを選ぶだけ」という3ステップの引き算の設計に特化させ、完全無料で提供する。

# 技術的にこだわりたい・工夫したい点（無理ない範囲で）

- 丁寧なDB設計とリレーション（中間テーブル）の構築：「食材」と「レシピ」を繋ぐ多対多の中間テーブル（使用量を含む）を綺麗に設計し、データの整合性を保ちます。

# なぜ自分がこれを作るのか（動機・原体験）

- 私の働いている店では、経営が苦しい時期もあり、オーナーが複数回変わっている。引き継ぎもうまくいかず、その度にやるべき作業が山のようにあった。食材の管理を任されていたこともあり、効率的で、簡単なシステムを作りたいと思った。
